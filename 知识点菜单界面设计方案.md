# 知识点菜单界面设计方案

## 一、三个选题模式的核心特质分析

### 1.1 中考配比模式（zhongkao）
**核心特质**：快速、自动化、考情导向
- ✅ **快速**：用户期望一键或两步完成选择
- ✅ **自动化**：系统根据年份套餐自动选择知识点
- ✅ **考情导向**：需要显示考频、年份、地区标签帮助用户理解选择依据
- ✅ **数据驱动**：基于真实真题数据，用户信任度高

**用户心理**：信任系统，希望快速获得符合中考标准的题目组合

### 1.2 专题模式（topic-middle）
**核心特质**：快速、聚焦、智能均分
- ✅ **快速**：选择专题后立即生成，减少操作步骤
- ✅ **聚焦**：针对特定语法大点深入练习
- ✅ **智能均分**：自动将题目分配到子知识点，用户无需手动分配
- ✅ **可视化**：需要清晰展示专题下的子知识点结构

**用户心理**：希望快速选择一个大专题，系统自动处理细节

### 1.3 自选模式（custom-middle）
**核心特质**：精准、灵活、可微调
- ✅ **精准**：用户需要精确选择到具体知识点（三级菜单）
- ✅ **灵活**：可跨专题、跨分类自由组合
- ✅ **可微调**：支持调整每个知识点的题目数量
- ✅ **信息完整**：需要显示考频、年份等信息辅助决策

**用户心理**：希望完全掌控，精准选择每个知识点

## 二、三级菜单结构分析

### 2.1 菜单层级
```
一级菜单（2个）：词法、句法
  ↓
二级菜单（14个）：名词、代词、形容词与副词、动词、介词、冠词、数词、连词、
                 句子成分与基本句型、动词时态、被动语态、主谓一致、复合句、特殊句式
  ↓
三级菜单（具体知识点）：如"名词所有格"、"人称代词"、"一般过去时"等
```

### 2.2 菜单特点
- **一级菜单**：分类标签，用于知识体系导航（词法 vs 句法）
- **二级菜单**：语法大点，是专题模式的选择目标
- **三级菜单**：具体知识点，是自选模式的选择目标

## 三、不同模式的菜单界面设计

### 3.1 中考配比模式菜单设计

1. **最小化用户操作**：只需选择年份套餐，系统自动完成
2. **考情信息突出**：显示考频、年份、地区标签
3. **结果可视化**：清晰展示系统自动选择的知识点

│  系统将根据以下规则自动选择：         │

│  • 语态：1题（必考）                 │
│  • 代词：1题                        │
│  • 介词：1题                        │
│  • ...（显示配比规则）               │
│                                      │
│  [生成题目] ← 一键生成               │
└─────────────────────────────────────┘

生成后显示：
┌─────────────────────────────────────┐
│  已选择的知识点（基于2025年真题）：   │
│                                      │
│  【词法】                            │
│  • 代词-物主代词 ⭐⭐⭐ 2025        │

- **菜单层级**：不显示完整菜单树，只显示结果
- **操作步骤**：年份选择 → 一键生成 → 显示结果
- **信息展示**：考频星星、年份标签、地区标签（云南）
- **数据来源**：基于用户提供的真题数据标注考频和年份

## 五、实现建议

### 5.1 数据准备
1. **建立三级菜单数据**：按照用户提供的菜单结构建立完整的数据模型
2. **真题数据映射**：将45道真题映射到对应的三级知识点
3. **考频计算**：根据真题数据计算每个知识点的考频（⭐⭐⭐）
4. **年份标注**：标注每个知识点在哪些年份考过

### 5.2 界面组件设计
1. **分组标签组件**：一级菜单作为不可点击的分组标签
2. **专题选择组件**：二级菜单作为专题模式的选择项
3. **知识点选择组件**：三级菜单作为自选模式的选择项
4. **考情标签组件**：显示考频、年份、地区标签

### 5.3 交互优化
1. **中考配比模式**：年份选择 → 一键生成，无需显示菜单树
2. **专题模式**：二级菜单直接可见，显示子知识点预览
3. **自选模式**：三级菜单平铺展示，支持筛选和搜索

### 5.4 性能优化
1. **懒加载**：自选模式下，按需加载知识点详情
2. **缓存机制**：缓存考情数据和题目数据
3. **虚拟滚动**：如果知识点数量多，使用虚拟滚动优化性能

## 六、关键设计决策

### 6.1 菜单层级的使用
- **一级菜单（词法/句法）**：仅作为视觉分组标签，不可选择
- **二级菜单（名词/代词/时态）**：专题模式的选择目标
- **三级菜单（具体知识点）**：自选模式的选择目标

### 6.2 考情信息的展示
- **考频星星**：⭐⭐⭐（根据真题出现次数计算）
- **年份标签**：2023, 2024, 2025（显示考过的年份）
- **地区标签**：云南（固定显示，因为数据来源是云南中考）

### 6.3 操作流程优化
- **中考配比**：2步（选择年份 → 生成）
- **专题模式**：3步（设置题型 → 选择专题 → 预览）
- **自选模式**：4步（设置题型 → 筛选 → 选择知识点 → 预览）

## 七、避免后期推翻的关键点

### 7.1 数据结构设计
- ✅ 使用三级菜单结构，支持不同模式的不同选择粒度
- ✅ 考情数据独立存储，便于更新和维护
- ✅ 题目数据与知识点分离，便于扩展

### 7.2 组件化设计
- ✅ 分组标签组件、专题选择组件、知识点选择组件独立
- ✅ 考情标签组件可复用
- ✅ 支持不同模式的组合使用

### 7.3 扩展性考虑
- ✅ 支持新增知识点（三级菜单）
- ✅ 支持新增真题数据
- ✅ 支持不同地区的考情数据（当前是云南，未来可扩展）

### 7.4 用户体验一致性
- ✅ 三个模式使用统一的数据源和考情标注
- ✅ 界面风格保持一致
- ✅ 操作逻辑清晰，符合用户预期

---

## 八、总结

根据三个选题模式的核心特质，知识点菜单的设计应该：

1. **中考配比模式**：不显示菜单树，只显示年份选择和生成结果
2. **专题模式**：二级菜单作为选择目标，一级菜单作为分组标签
3. **自选模式**：三级菜单作为选择目标，一级菜单作为分组标签，二级菜单可折叠

这样的设计既满足了不同模式的核心需求，又保持了数据结构和界面的一致性，避免了后期推翻重来的风险。
