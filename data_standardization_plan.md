# 数据标准化计划

## 🎯 目标
- 统一前后端命名规范
- 均衡各语法点题目数量
- 建立标准化的分类体系

## 📋 标准化规范

### 1. 命名规范
- **前端显示名称** = **数据库分类名称**
- 使用中文，简洁明了
- 避免歧义，保持一致性

### 2. 分类粒度标准
- **一级分类**：名词、动词、形容词、副词、介词、连词、冠词、代词、从句
- **二级分类**：具体的语法点，如"一般过去时"、"被动语态"等

### 3. 题目数量标准
- **每个语法点**：15-25题（理想20题）
- **最少**：10题
- **最多**：30题
- **超出范围**：重新分配或合并

## 🔧 实施步骤

### Phase 1: 数据清理（3天）
1. 分析现有题目分布
2. 识别重复和缺失
3. 制定重新分配方案

### Phase 2: 分类重构（4天）
1. 重新设计分类体系
2. 批量重命名数据库分类
3. 更新前端显示名称

### Phase 3: 题目重新分配（3天）
1. 将题目按新分类重新分配
2. 补充缺失的题目
3. 删除重复题目

### Phase 4: 测试验证（2天）
1. 全面测试所有语法点
2. 验证题目数量均衡
3. 确认前后端一致性

## 📊 预期结果
- 语法点数量：约50个标准语法点
- 总题目数量：1000-1250题
- 每个语法点：20题左右
- 前后端100%匹配，无需映射表
